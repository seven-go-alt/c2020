## 视觉反馈驱动型调优的黑盒与瓶颈

在精品游戏的工业化制作管线中，除去资产生产与代码研发外，**策划调优（Design Tuning）往往占据了极高的时间成本与迭代比重**。这一环节类似于“工匠式”的精细打磨，高度依赖资深策划的专家经验，是决定游戏最终品质的"最后一公里"。无论是关卡心流的把控、场景空间的布局，还是战斗中敌兵的站位美感，都难以仅凭静态数值模拟或理论推演得到定论。**这些设计必须依赖高频次的实机验证，通过多轮迭代逐步逼近最佳体验。**

这类调优链路的核心痛点在于：**调优决策呈现明显的“黑盒”特征，且强依赖对运行时画面（Runtime Visuals）的感性认知和主观评估**。策划需要通过视觉反馈捕捉动态变化——包括节奏张力、空间关系、视觉冲击力、压力与反馈强度等——再将这些难以量化的主观体验，“翻译”为具体的参数修正方向。我们可以将此类工作定义为**视觉反馈驱动型调优（Visual Feedback Driven Tuning）**，并通常遵循以下典型工作流：

1. **参数配置（Configuration）**：在编辑器中基于设计意图设定关卡、AI 行为树或场景元数据。
    
2. **实机装载（Runtime Loading）**：编译代码、烘焙资源并启动引擎，使内容进入 Runtime 可交互状态。
    
3. **人工试玩（Playtesting）**：策划亲身投入游戏，通过实时渲染画面获取视觉与操作的双重反馈。
    
4. **评估修正（Evaluation & Adjustment）**：基于主观体验判断优劣，定位问题并回退至编辑器调整参数。
    
5. **循环迭代（Iterative Loop）**：重复上述流程，直至体验达到验收标准。
    

![视觉反馈驱动型调优工作流](https://kmpvt.pfp.ps.netease.com/file/698c4af48b6a8550318444abmXjxE1gj01?sign=Q1tLZqpLKzYjbUgy09z4w8_ixrA=&expire=1775208152&type=image/jpeg "视觉反馈驱动型调优工作流")视觉反馈驱动型调优工作流

这个流程本质上是一种**高频手动试错（Trial and Error）的工作流**，也被业内戏称为“手搓”。从技术工业化的角度审视，其局限性主要体现在：

- **高昂的时间沉没成本**：效率受限于单次构建与加载耗时。若修改一个数值需等待数分钟才能看到效果，创作心流将被严重打断。
    
- **非确定性与主观偏差**：结果高度依赖主观审美，缺乏客观的数据度量标准，导致调优成果难以复用或标准化。
    
- **自动化与回归缺失**：由于依赖“人眼”判断，这种调优难以接入自动化测试管线。每次版本更新或底层代码变动，都可能导致精心调优的手感崩坏，且只能依靠人工再次全量回归。
    

## EQS 调优实践

[场景查询系统（Environment Query System，EQS）](https://dev.epicgames.com/documentation/zh-cn/unreal-engine/environment-query-system-in-unreal-engine)是虚幻引擎5（UE5）AI 系统的一个功能，可将其用于从环境中收集数据，可以作为行为树决策节点的底层支撑。EQS 由三类核心节点构成：

- 生成器（Generator）：生成位置或 Actor，其被称为项目（Item）、实际上将被测试和加权。
    
- 情境（Contexts）：为各种测试和生成器提供引用的框架。
    
- 测试（Tests）：确定环境查询如何决定来自生成器的哪个项目是最佳选择。
    

简单来说，EQS 的核心逻辑遵循“生成-测试-评分”的管线：情境节点（Context）提供必要的上下文，生成器节点（Generator）生成一组候选采样点（Items），然后测试节点（Tests）对这些候选项执行一组测试（例如，视线检测、距离判定、拥挤约束等）以完成过滤与加权打分，最终返回“最佳 Item”。

![EQS 示意图](https://kmpvt.pfp.ps.netease.com/file/69819ae0a6c9122fcc5649ceX50yX57F01?sign=CIXxJmjwYFx05GBwJZW096A8ObQ=&expire=1775208152&type=image/png&fop=imageView/6/f/webp/q/50 "EQS 示意图")EQS 示意图

在《归唐》项目中，广泛使用 EQS 作为敌兵 AI 的基础能力。**为了追求“电影级武打呈现”的品质目标，我们不仅要求敌兵站位满足基本约束（纵向近/中/远圈层感、横向左/中/右分布均匀、单位间距合理、避免重叠与卡位），还期望其具备更强的“镜头质感”（例如形成阵型氛围，营造镜头张力与视觉冲击）。**然而，这种高品质要求显著抬高了 EQS 的策略上限，也使其调优过程面临严峻挑战：

### 1. 参数空间爆炸

EQS 的配置本质上是一个极高维度的参数矩阵。一个典型的查询涉及生成器形态（Grid/Donut/Circle）、多重测试项（Trace/Distance/Dot Product）、评分公式（Linear/Inverse/Square）、归一化参考系及权重（Weight）。 **这些参数高达几十维，且相互耦合。参数组合呈指数级增长，微小的数值变动可能导致完全不同的决策结果，因此几乎没有人工遍历搜索最优解的可能性。**

### 2. “视觉表现—底层数值”的映射鸿沟

EQS 调优本质上是在感性体验与数学机制之间做对齐：策划在实机中观察的是具体的视觉现象（如“敌兵走位经常卡角落”、“多人围攻时容易挤成一团”、“敌兵站位过于分散没有压迫感”等等），而可调手段却是抽象参数。**由于输入端（视觉）到输出端（数值）中间缺少直观的诊断桥梁和统一的量化标尺，调优往往只能凭感觉“猜”，流程最终退化为一种“看画面调公式”的盲盒式反推。**

### 3. 经验壁垒

**EQS 调优具有极高的学习门槛，它不仅要求掌握客观的游戏引擎知识（例如理解 Donut Generator 的环形采样原理），更极度依赖专家级的隐性知识（例如只有经验丰富的策划才能深刻理解不同测试项组合后的非线性化学反应）。**因此，许多关键经验只能由资深策划凭借长期试错中形成“直觉”，而这种经验难以沉淀为标准化的资产或文档，导致对专家经验的依赖度极高，新人难以接手。

### 4. 自动化

当前 EQS 调优仍普遍停留在“手动配置 → 实机运行 → 肉眼观察 → 凭经验修正”的手工坊模式。**由于缺乏基于可量化数据指标的自动化工具，当关卡规模扩展至海量内容时，调优所需的人力与时间成本将迅速攀升，难以支撑高频迭代诉求。**

## EQS Agent

为了突破手动调优范式的效率瓶颈，我们将 EQS 调优过程建模为马尔可夫决策过程（Markov Decision Process，MDP），抽象为**环境（Environment）—智能体（Agent）—奖励（Reward）**三个核心要素，并引入**Retrieval-Augmented Generation（RAG，检索增强生成）和 Human-in-the-Loop（HITL，人机协同）**机制，构建了一套可闭环、可解释的自动化调优系统。

在该框架下，**智能体在参数空间内选择微调操作（Action），直接对 EQS 配置文件进行局部调整**（如修改 Donut Generator 半径），并在真实游戏包体中触发“参数装载—实机运行—自动试玩”的全流程自动化跑测。**评估模型基于实机表现数据（State，包含底层 EQS 日志与渲染画面等多模态信号）反馈量化奖励（Reward）**，驱动智能体在“试探—反馈—学习”的闭环中持续迭代，逐步收敛至更优配置。为确保系统兼具探索广度与决策深度，我们进一步引入了 RAG 与 HITL 双重机制：**RAG 负责将文档知识与历史调优经验结构化管理和检索，为 Agent 提供实时的上下文指引**；**HITL 允许策划作为监督者，随时介入干预（如约束边界、修正评估准则）**，确保调优过程不仅高效，而且可解释、可回溯、安全落地。

![EQS Agent](https://kmpvt.pfp.ps.netease.com/file/69819c36e22f626f82f0a834IJFeXIqA01?sign=vrlrvhwhpZmJ65DxKuhLcG0y21Q=&expire=1775208152&type=image/jpeg&fop=imageView/6/f/webp/q/50 "EQS Agent")EQS Agent

### 环境：跑图机

在传统调优工作流中，“参数配置—实机装载—人工试玩”的物理链路不仅人力成本高昂，且极易受人为操作的随机性干扰。**跑图机（Automated Runner）的核心使命便是将这一过程标准化与自动化：它作为一个可运行客户端实例，负责接收 Agent 下发的 EQS 配置文件，自动挂载参数、重置关卡状态，并执行跑测。**

为确保评估基准的一致性，跑图机执行的是严格的确定性遍历流程：通过预设的路线驱动角色在场景中按固定路线移动，消除人工试玩带来的操作噪声。在此过程中，系统集成脚本会同步执行多模态采集，捕获运行时的底层数据（EQS Debug Log）与表层视觉表现（实机渲染画面/游戏视频）。最终，跑图机将输出一份标准化的“评估数据包”，包含 EQS 配置文件、EQS 日志以及游戏视频，为下游的评估模型提供精确、可复现的输入。

### 奖励：双层漏斗多模态评估模型

建立奖励函数的核心挑战，**在于将“好不好看、合不合理”这类主观感受转化为计算机可度量的梯度信号**，用于判定当前结果优劣，并为 Agent 提供明确的优化目标。为此，我们构建了一个**两层漏斗式的多模态评估模型，从“满足下限”到“追求上限”逐级收敛**。

#### 第一层：基于规则驱动的缺陷检测度量（Defect Metrics）—— “先确定不犯错”

Agent 直接解析 EQS 日志中的底层数据，对站位表现进行硬性约束检查，快速过滤掉不可接受的劣质解。策划可通过参数化配置灵活定义约束边界，例如在“纵向圈层感”规则中，设定近圈半径为 0–3m 且人数上限为 1 人。一旦检测到贴脸拥挤或站位失衡等违规行为，即判定为无效解，从而大幅降低无效迭代的成本。

#### 第二层：基于 VLM 驱动的美观度度量（Aesthetic Metrics）—— “再追求精彩”

在通过缺陷检测的基础上，该层聚焦于评估阵型的氛围张力、构图秩序与视觉压迫感，进一步评估站位是否具备更强的“镜头质感”。这一层以 VLM 驱动为核心。考虑到美学判断天然主观且抽象，我们设计了一套 CoT（Chain-of-Thought）引导框架，用于结构化约束 VLM 对实机视频中的视觉信息进行分析，并且策划可将期望的审美偏好与评估准则以 prompt 的形式注入模型。最终，VLM 输出美观度分数及可解释的描述性评估文本，为后续调参提供可追溯依据。

> 在技术落地层面，VLM 的选型具备灵活性：在合规允许的前提下可调用 Gemini 3 Pro 等头部云端模型以获取最佳理解力，若涉及严格的数据隐私要求，亦可私有化部署 Qwen-VL 等模型。

### 智能体：调优 AI Agent

调优 AI Agent 是整套闭环的调度中枢，其本质是一个由 LLM 驱动的“虚拟策划”。它不仅具备对 EQS 参数的理解能力，**更能自主编排并调用底层工具链，像科学家一样执行严谨的“假设—实验—论证”循环**。其核心工作流包含四个紧密耦合的步骤：

1. **实验规划（Plan）**：基于当前的调优目标与硬性约束，Agent 结合对前序上下文的理解，提出具体的实验假设。它会明确本轮迭代的 Action 集合——包括调整的参数维度、具体的数值幅度以及预期的表现变化。
    
2. **实机验证（Execute）**：Agent 将生成的实验计划转化为标准化的 EQS 配置文件，并调度跑图机在真实游戏包体中执行自动化跑测。这一步将抽象的数值策略转化为具象的运行表现，生成标准化实验数据包。
    
3. **诊断评估（Evaluate）**：启用双层漏斗多模态评估模型，对实验数据进行全方位“体检”。Agent 不仅关注最终的分数（Reward），更会深入分析缺陷成因与美学表现，从数据中归纳出当前版本的主要瓶颈（如“近战圈过于拥挤”或“空间分布率不足”）。
    
4. **检索知识（Retrieve）**：针对诊断出的具体问题，Agent 调用 RAG 引擎检索知识库与经验库。通过查询参数的底层语义、官方配置要点以及历史沉淀的 Good/Bad Case，它将关键知识注入决策上下文。这不仅为下一轮实验提供了可执行的解决方案，也确保了每一次迭代都是基于“前人经验”的理性修正，而非盲目试错。
    

### RAG：知识库和经验库

**EQS 调优过程中沉淀的大量知识与经验，这些隐性资产是团队最宝贵却也最易流失的资产，因此如何将它们结构化管理是闭环体系中不可或缺的一环。**为此，我们引入 RAG 机制，构建了由 “知识库” 与 “经验库” 双轮驱动的资产管理层，将离散的信息转化为可检索、可推理的结构化能力。

- **知识库（Knowledge Base）**：旨在沉淀客观的引擎与研发知识。 我们基于 GraphRAG 技术搭建了文档自动化解析链路，能够将项目研发文档、Confluence 维基以及 UE 官方手册等异构数据进行结构化抽取，并构建为具备语义关联的知识图谱。这使得 Agent 在调优过程中，不仅能检索到单一参数的定义，更能通过图谱的关联推理能力，快速获取适用场景与配置最佳实践，消除了查阅文档的信息孤岛。
    
- **经验库（Experience Base）**：则专注于捕捉动态的调优轨迹与专家直觉。 鉴于 EQS 调优的高度“黑盒”特性，静态文档往往难以复原复杂的决策逻辑。因此，我们完整记录了人机协同的全过程——包括 Agent 的探索动作、环境反馈、评估结果以及至关重要的 Human-in-the-loop 人工干预点。这种机制不仅利用专家介入加速了收敛，更将稍纵即逝的调试灵感转化为标准化的 Good/Bad Case 样本集。这些积累的经验数据为后续的模型训练、策略更新以及跨关卡的问题诊断提供了坚实的数据支撑，真正实现了从“个人经验”到“团队资产”的转化。

随后，Agent 将自动调用评估模型对数据包进行解析，并提供多维度的可视化分析报告 。其中，单帧详情分析（微观）： 针对每一个关键帧，提供“三位一体”的可视化复盘。包含游戏实机截图、基于 EQS 日志还原的 2D/3D 场景雷达图，以及各评估模型的具体打分详情，帮助用户精准定位问题；全局数据概览（宏观）： 汇总页面展示本次数据包中所有帧的整体评估情况。通过统计图表宏观呈现测试数据的通过率、缺陷分布及美学评分趋势，为调优决策提供全局依据。

1. **【分析节点】**Agent 调用数据分析工具链对上述评估结果进行归因诊断，自动输出问题清单。**此处，支持深度的人机协作**，用户可点击“编辑”按钮，对 Agent 生成的“具体表现”、“分析理由”及“影响程度”进行人工校准与补充，并且通过右侧勾选框，用户可自主决定是否将该问题纳入后续的自动优化流程，确保优化的针对性。

1. **【优化节点】**用户提交问题清单后，Agent 将基于 RAG 技术检索关联参数知识及历史调优案例，综合推理出包含参数名称、数值对比（当前值/建议值）、修改理由及预期效果的详细调优方案。同样，此处支持深度的人机协作，用户可对 Agent 的建议进行自由编辑、拒绝或勾选采纳；确认无误后，点击右上角“确认参数并更新资产”按钮，系统将自动生成新的配置文件覆盖原参数，**并将本次优化的决策逻辑同步收入经验库以供后续参考**，实现 Agent 的自我迭代与持续进化。
